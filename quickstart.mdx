This guide walks you through launching the CLI, entering your first task, browsing a website, and reviewing the generated API client.

## Prerequisites

Make sure you have completed [installation](/installation) and have:
- `reverse-api-engineer` installed (`uv tool install reverse-api-engineer`)
- Playwright Chromium installed (`playwright install chromium`)
- `ANTHROPIC_API_KEY` set in your environment

## Your first API client

<Steps>
  <Step title="Launch the CLI">
    Run the CLI from any directory:

    ```bash
    reverse-api-engineer
    ```

    The interactive prompt appears with the current mode shown in brackets. The default mode is **agent**.

    ```
    [agent] > 
    ```

    <Tip>
      Press **Shift+Tab** to cycle through modes: agent → manual → engineer → collector. The mode indicator updates in real time.
    </Tip>
  </Step>

  <Step title="Switch to manual mode and enter a task">
    Press **Shift+Tab** once to switch to manual mode, then type a description of what you want to capture:

    ```
    [manual] > fetch all apple jobs from their careers page
    ```

    The CLI will ask for a starting URL (optional):

    ```
    Starting URL (optional, press Enter to skip):
    > https://jobs.apple.com
    ```
  </Step>

  <Step title="Browse the website">
    A Chromium browser opens with HAR recording active. Navigate and interact normally:

    - Search for jobs
    - Click through listings
    - Apply any filters you want the API client to support

    The browser records every network request in the background.

    <Note>
      The browser uses stealth mode to minimize bot detection. Browse naturally — don't rush through interactions.
    </Note>
  </Step>

  <Step title="Close the browser">
    When you're done exploring the site, close the browser window. The CLI detects the closure and starts AI analysis automatically:

    ```
    ✓ Browser closed — HAR captured
    ✓ Analyzing 47 requests...
    ✓ Identified 3 API endpoints
    ✓ Generating api_client.py...
    ```

    The AI filters noise (analytics, ads, CDN requests) and focuses on the relevant API calls.
  </Step>

  <Step title="Review your generated files">
    When generation completes, find your files in two locations:

    ```
    ✓ Scripts saved to: ./scripts/apple_jobs_api/
    ```

    ```
    ./scripts/apple_jobs_api/
    ├── api_client.py      # Full API implementation
    ├── README.md          # Endpoint documentation
    └── example_usage.py   # Runnable usage examples
    ```

    Open `api_client.py` to see the generated client with type hints, docstrings, and error handling.
  </Step>
</Steps>

## Slash commands inside the CLI

While the CLI is running, you can use these slash commands:

| Command | Description |
|---------|-------------|
| `/settings` | Configure model, SDK, output language, agent provider |
| `/history` | View past runs with costs and timestamps |
| `/messages <run_id>` | View detailed message logs for a run |
| `/help` | Show all available commands |
| `/exit` | Quit the CLI |

```
[manual] > /history

  run_id           date              cost     prompt
  abc-123-def      2026-03-17 14:32  $0.0234  fetch all apple jobs...
  xyz-789-ghi      2026-03-16 10:15  $0.0189  scrape product catalog...
```

## Try agent mode

Agent mode lets the AI browse autonomously — no manual navigation needed:

```bash
reverse-api-engineer
```

```
[agent] > find the top 10 job listings on LinkedIn for Python engineers in San Francisco
```

The agent controls the browser, navigates the site, and captures the relevant traffic without any input from you.

## Re-engineer a capture

If you want to regenerate the client with different instructions, switch to engineer mode and reference your run ID:

```
[engineer] > @id abc-123-def generate a more detailed client with pagination support
```

See [Engineer mode](/modes/engineer) for all the engineer mode tags.
