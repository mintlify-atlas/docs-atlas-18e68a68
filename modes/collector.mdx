---
title: "Collector mode"
description: "AI-powered web data collection with automatic JSON and CSV export."
---

Collector mode uses an AI agent to search the web, browse pages, and gather structured data based on a natural language prompt. The results are automatically exported to JSON and CSV files.

Unlike the other modes, collector mode does not capture HAR files or generate API clients. It is purpose-built for extracting and saving structured data.

## How to use it

Press **Shift+Tab** until `[collector]` appears in the prompt, then describe the data you want to collect:

```
[collector] > Find 3 JS frameworks
```

```
[collector] > Collect top 10 Python libraries with name, description, and GitHub stars
```

The agent uses `WebFetch`, `WebSearch`, and file tools to autonomously search, browse, and save items as it finds them. When collection is complete, the output files appear in two locations:

- `~/.reverse-api/runs/collected/{folder_name}/` — permanent storage
- `./collected/{folder_name}/` — local copy

## Example

```bash
$ reverse-api-engineer
[collector] > Find 3 JS frameworks

# Agent autonomously searches and collects data
# Data saved to: ./collected/js_frameworks/
```

The resulting `items.json` contains structured records like:

```json
[
  {
    "name": "React",
    "description": "A JavaScript library for building user interfaces",
    "website": "https://react.dev",
    "source_url": "https://github.com/facebook/react"
  }
]
```

## Configuration

Collector mode uses the `collector_model` setting. The default is `claude-sonnet-4-6`. Change it by editing `~/.reverse-api/config.json` directly:

```json
{
  "collector_model": "claude-sonnet-4-6"
}
```

<Note>
  Collector mode requires `ANTHROPIC_API_KEY` — it always uses the Claude Agent SDK regardless of your SDK setting.
</Note>
