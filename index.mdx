## What is Reverse API Engineer?

Reverse API Engineer is a CLI tool that captures browser traffic in HAR (HTTP Archive) format and uses Claude AI to automatically generate production-ready API clients. Browse any website, close the browser, and get clean, typed, documented Python (or JavaScript/TypeScript) code that replicates those API calls.

```bash
# Install
uv tool install reverse-api-engineer

# Launch
reverse-api-engineer

# In the CLI prompt:
> fetch all apple jobs from their careers page
# Browser opens → navigate → close → AI generates api_client.py
```

## How it works

<Steps>
  <Step title="Capture traffic">
    A Playwright browser launches with HAR recording enabled and stealth mode active. You browse normally — or let an AI agent browse for you.
  </Step>
  <Step title="Close the browser">
    When you're done interacting with the site, close the browser. All network traffic has been recorded to a `.har` file.
  </Step>
  <Step title="AI generates client code">
    Claude analyzes the HAR file, identifies API endpoints, authentication patterns, and data structures, then writes a production-ready API client.
  </Step>
  <Step title="Use your generated client">
    The generated scripts appear in `./scripts/{name}/` with an `api_client.py` (or `.js`/`.ts`), `README.md`, and example usage file.
  </Step>
</Steps>

## Example output

After capturing traffic from a job board, Reverse API Engineer produces:

```python api_client.py
import httpx
from typing import Optional

class AppleJobsAPI:
    """API client for Apple Jobs (jobs.apple.com)."""

    BASE_URL = "https://jobs.apple.com/api/role/search"

    def __init__(self, session: Optional[httpx.Client] = None):
        self.session = session or httpx.Client()

    def search_jobs(
        self,
        query: str,
        location: Optional[str] = None,
        page: int = 1,
        page_size: int = 20,
    ) -> dict:
        """Search Apple job listings.

        Args:
            query: Search query string
            location: Filter by location name
            page: Page number (1-indexed)
            page_size: Results per page

        Returns:
            Dict with searchResults list and totalRecords count
        """
        payload = {
            "query": query,
            "filters": {"range": {"START": page - 1, "END": page_size}},
        }
        if location:
            payload["filters"]["locations"] = [{"type": "CITY", "code": location}]

        response = self.session.post(self.BASE_URL, json=payload)
        response.raise_for_status()
        return response.json()
```

<Tip>
All generated clients include type hints, docstrings, error handling, and authentication support out of the box.
</Tip>
